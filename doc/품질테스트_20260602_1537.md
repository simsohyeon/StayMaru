## ❌ 안 되는 부분

### 🔴 1. 빅데이터 "활용신청 안내" UX가 작동하지 않음 — 실제 코드 버그
- **현상:** `TarRlteService1`(연관추천)은 HTTP **500**, `DataLabService`(방문자통계)는 HTTP **403**을 평문으로 반환(활용신청 미완료 상태).
- **원인:** `frontend/src/api/bigdata.ts`의 `callBigData`에 **`validateStatus` 설정이 없어** axios 기본값(2xx만 통과)이 4xx/5xx에서 곧바로 `throw` → `catch` 블록의 `status: 'error'`로 떨어짐. 그 결과 평문을 `not-subscribed`로 분류하는 로직(108~114행)과 resultCode 분류(116~119행)는 **현재 게이트웨이 동작에서 절대 실행되지 않는 죽은 코드**.
- **영향:** Insights 페이지가 의도한 *"활용신청하면 자동으로 채워집니다"* 안내(`bigdata.activateBody`) 대신 일반 *"데이터 없음"*(`bigdata.empty`) 문구 노출. 주석에 명시된 *"데이터 활용도 심사 포인트"* showcase 의도가 깨짐. (앱 크래시는 없고 활용신청 CTA 링크 자체는 표시됨 → 심각도 中)
- **권고 수정:** axios 호출에 `validateStatus: () => true` 적용 후 상태코드 기반으로 not-subscribed/error 분기, 또는 catch 내부에서 `err.response.status`(401/403/500)를 not-subscribed로 분류.

### 🔴 2. 빅데이터 기능(연관추천·방문자랭킹) 자체가 빈 화면 — 운영 신청 미완료
- TarRlteService1 / DataLabService 모두 **공공데이터포털 무료 활용신청 미완료** 상태로 추정(키는 KorService2엔 유효하나 두 서비스엔 403/500).
- 신청 전에는 PlaceDetail의 "함께 찾은 곳" 섹션이 안 뜨고, Insights의 방문자 랭킹/주목 관광지가 비어 있음. (코드상 graceful 숨김은 동작하나, #1 때문에 안내 문구 부정확)
