## ❌ 안 되는 부분

### 🔴 1. DataLabService(방문자 통계)가 파라미터 오류로 작동 안 함 — 신규 발견
- **현상:** 이전엔 HTTP 403(미구독)이었으나 이제 **HTTP 200**으로 응답 — 서비스 자체는 열림. 그러나 모든 요청이 `resultCode:"10" INVALID_REQUEST_PARAMETER_ERROR`로 거부됨(4회 일관, 간헐 아님).
- **검증한 파라미터 조합 (전부 거부):**
  - `signguCd=47130` 단독 (앱 현재 방식) → `INVALID_REQUEST_PARAMETER_ERROR(signguCd)`
  - `areaCd=47 + signguCd=47130` → `INVALID_REQUEST_PARAMETER_ERROR(areaCd)`
  - `areaCd=47` 단독 → `INVALID_REQUEST_PARAMETER_ERROR(areaCd)`
  - `metcoRegnVisitrDDList` 오퍼레이션 변형 → `INVALID_REQUEST_PARAMETER_ERROR(areaCd)`
- **영향:** Insights 페이지의 **시군별 방문자 랭킹이 데이터를 채우지 못함**. `bigdata.ts`가 resultCode "10"을 `not-subscribed`로 매핑(135행)하므로 화면엔 "활용신청 안내" 카드가 뜨지만, 실제로는 서비스가 이미 구독돼 있어 **안내 문구도 부정확**(미신청이 아니라 파라미터 스펙 불일치).
- **원인 추정:** `locgoRegnVisitrDDList`의 필수 파라미터 명/조합이 앱 구현(`signguCd` 단독, `frontend/src/api/bigdata.ts:310-315, 331-336`)과 불일치. 공공데이터포털 DataLabService 명세 재확인 필요.

### 🟠 3. 전국문화축제표준데이터 API 불안정(Flaky) — 업스트림 이슈
- 동일 요청 연속 호출 시 **try1=HTTP 200(1599B, 1.5s), try2=HTTP 500(0B, 0.07s)** 로 간헐 실패.
- `standardFestival.ts`는 1·2페이지를 `Promise.all`로 동시 요청 → 한 페이지라도 500이면 전체 fetch 실패(catch 후 빈 배열 폴백). 따라서 **축제 목록이 간헐적으로 비어 보일 수 있음**. (data.go.kr 업스트림 측 불안정, 앱 코드 결함 아님 — 단 재시도 로직 부재)

### 🟠 4. 로컬 `npm run build` 크래시 — 환경 이슈 (제품 결함 아님)
